1. Decision Modeling

Decision-making in agroecology often requires navigating trade-offs, for example between productivity and biodiversity. Decisions are further complicated by system complexity and uncertainty, i.e. the impacts of climate variability or market fluctuations. In this section, we will explore why decision modeling matters for agroecology, how it can guide sustainable land management, and what tools can help us make informed decisions.

6. The Reference Model: monte_carlo.R

monte_carlo.R is the foundational model. It defines five uncertain variables, a simple decision model function, and a single call to mcSimulation(). All inputs are treated as annual values — there is no explicit time horizon. The discount() function is applied to a single number, so it converts an annual net benefit into a present value using the discount rate.

Workflow

  1. Build an input_estimates data frame with probability distributions for each variable.
  2. Write a model_function() that computes outcomes from those variables.
  3. Call mcSimulation() to sample from the distributions and run the model thousands of times.
  4. Plot and interpret the resulting distributions.

Step 1: Define the input estimates

The table below covers exactly five variables. posnorm draws from a positive normal distribution bounded by lower and upper. const fixes the variable at its lower value with no uncertainty.

input_estimates <- data.frame(
  variable = c("sheep_income", "sheep_cost", "apple_income", "apple_cost", "discount_rate"),
  lower = c(3000, 1000, 30000, 15000, 10),
  median = NA,
  upper = c(5000, 2500, 60000, 30000, 10),
  distribution = c("posnorm", "posnorm", "posnorm", "posnorm", "const"),
  label = c("Income from sheep (euro/year)", "Cost of sheep (euro/year)",
            "Income from apple (euro/year)", "Cost of apple (euro/year)", "Discount Rate")
)

Step 2: Write the model function

Each call to model_function() receives one draw of each variable from the distributions above. The function computes the net outcome for the agroforestry system, the treeless sheep system, and the difference (the decision benefit). discount() converts each annual result into a Net Present Value (NPV).

model_function <- function() {
  AF_income  <- sheep_income + apple_income
  AF_cost    <- sheep_cost  + apple_cost
  AF_final_result  <- AF_income  - AF_cost
  sheep_only       <- sheep_income - sheep_cost
  Decision_benefit <- AF_final_result - sheep_only

  AF_NPV        <- discount(AF_final_result,  discount_rate = discount_rate, calculate_NPV = TRUE)
  NPV_sheep_only <- discount(sheep_only,       discount_rate = discount_rate, calculate_NPV = TRUE)
  NPV_decision   <- discount(Decision_benefit, discount_rate = discount_rate, calculate_NPV = TRUE)

  return(list(
    NPV_Agroforestry_System = AF_NPV,
    NPV_Treeless_System     = NPV_sheep_only,
    NPV_decision            = NPV_decision
  ))
}

7. Extending the Model with Time: monte_carlo_add.R

The model in Section 6 treats all inputs as single annual values — useful for a first pass, but it cannot capture how outcomes unfold over time. What if apple trees take 4–6 years to mature before producing any income? What if there are large upfront costs in year one that dwarf later annual costs? A single-year average glosses over all of that.

Here we extend the model to handle exactly that. Income and costs are tracked year by year across a full time horizon, and discount() processes a vector of yearly values to return a true multi-year NPV.

Key differences from monte_carlo.R

Feature monte_carlo.R monte_carlo_add.R
Number of variables 5 8
Time horizon None (single-year) n_years (e.g. 20 years)
Apple income delay None Zero until time_to_harvest
Establishment cost None cost_orchard_establishment in year 1
make_variables() helper No Yes — for testing a single draw
Input to discount() Scalar Year-vector (length n_years)

Workflow

  1. Add the make_variables() helper to sample one draw and inspect variable values.
  2. Extend the input_estimates table with the three new variables.
  3. Modify the model function to build year-vectors and apply the time-based rules.
  4. Test the model function with a single draw before running the full simulation.
  5. Run mcSimulation() as before.

Step 1: The make_variables() helper

make_variables() draws one sample from the estimate distributions and assigns each variable into the global environment. This lets you call model_function() interactively and inspect intermediate values — useful for debugging before running thousands of simulations.

make_variables <- function(est, n = 1) {
  x <- random(rho = est, n = n)
  for (i in colnames(x)) assign(i, as.numeric(x[1, i]), envir = .GlobalEnv)
}

Step 2: Extend the input estimates

Three variables are added: n_years and discount_rate are fixed constants; cost_orchard_establishment and time_to_harvest are uncertain. Notice time_to_harvest uses a unif (uniform) distribution because the uncertainty is bounded without a clear central tendency.

input_estimates_add <- data.frame(
  variable = c("sheep_income", "sheep_cost", "apple_income", "apple_cost",
               "discount_rate", "n_years", "cost_orchard_establishment", "time_to_harvest"),
  lower    = c(3000, 1000, 30000, 15000, 10, 20, 50000, 4),
  median   = NA,
  upper    = c(5000, 2500, 60000, 30000, 10, 20, 70000, 6),
  distribution = c("posnorm", "posnorm", "posnorm", "posnorm",
                   "const", "const", "posnorm", "unif"),
  label = c("Income from sheep (euro/year)", "Cost of sheep (euro/year)",
            "Income from apple (euro/year)", "Cost of apple (euro/year)",
            "Discount Rate", "Simulated Years",
            "Construction cost of orchard", "Time until fruit harvest")
)

Step 3: Write the time-aware model function

The model now builds a vector of length n_years for each income and cost stream. Apple income is set to zero for the first time_to_harvest years (the establishment phase), and the full orchard construction cost replaces the normal apple cost in year 1.

model_function_add <- function() {
  # time_to_harvest must be an integer for indexing
  time_to_harvest <- round(time_to_harvest, digits = 0)

  # Expand scalar inputs into year-vectors
  apple_income <- rep(apple_income, n_years)
  apple_cost   <- rep(apple_cost,   n_years)
  sheep_cost   <- rep(sheep_cost,   n_years)
  sheep_income <- rep(sheep_income, n_years)

  # No apple income during the establishment phase
  apple_income[1:time_to_harvest] <- 0

  # Upfront orchard establishment cost replaces year-1 apple cost
  apple_cost[1] <- cost_orchard_establishment

  AF_income        <- sheep_income + apple_income
  AF_cost          <- sheep_cost   + apple_cost
  AF_final_result  <- AF_income  - AF_cost
  sheep_only       <- sheep_income - sheep_cost
  Decision_benefit <- AF_final_result - sheep_only

  AF_NPV         <- discount(AF_final_result,  discount_rate = discount_rate, calculate_NPV = TRUE)
  NPV_sheep_only <- discount(sheep_only,        discount_rate = discount_rate, calculate_NPV = TRUE)
  NPV_decision   <- discount(Decision_benefit,  discount_rate = discount_rate, calculate_NPV = TRUE)

  return(list(
    NPV_Agroforestry_System = AF_NPV,
    NPV_Treeless_System     = NPV_sheep_only,
    NPV_decision            = NPV_decision
  ))
}

Step 4: Test with a single draw

Use make_variables() to populate the global environment and call the model function once before running the full simulation. This is a good debugging practice.

Optional: Pareto-Optimality (Advanced)

Now we will calcualte Pareto-Optimality. We will use the concept of the Pareto front to find the best trade-offs between two competing objectives. We can learn which solutions are optimal (better in at least one way, without being worse in another) and which solutions are dominated (we could improve at least one objective without making the other worse).

First we need to increase the model complexity a little by adding management options. We add stocking density (sheep per hectare), tree density (trees per hectare), proportion of tree area, and proportion of sheep area (1 - proportion_tree_area). Adjust these numbers to reflect your best estimate.
